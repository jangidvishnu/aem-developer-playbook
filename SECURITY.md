# Security Policy

## Supported versions

The live site and the default branch (`master`) are what we support for security fixes.

## Reporting a vulnerability

Please **do not** open a public issue for security-sensitive reports.

Contact the repository owner privately via GitHub ([@jangidvishnu](https://github.com/jangidvishnu)) using a private message, Security Advisories (if enabled on this repo), or the email listed on their GitHub profile.

Include:

- Description of the issue
- Steps to reproduce
- Impact assessment if known
- Whether you plan to disclose publicly and on what timeline

We will acknowledge receipt when we can and work on a fix or mitigation.

# Key Companies Migrating to AEM Cloud and Related Technologies

Our deep research identified several global organizations publicly adopting or migrating to **Adobe Experience Manager (AEM) as a Cloud Service** and related technologies (Edge Delivery, Universal Editor, GraphQL, Content/Experience Fragments, Adobe Experience Platform, etc.). For each company below we summarize their industry, type, location, India presence, careers link, technologies used, evidence of adoption (with citations), migration status, Edge Delivery adoption, typical roles, India hiring/visa info, and attractiveness for AEM engineers. The full dataset is compiled in the downloadable Excel/CSV files below.

- **E.ON** (Energy, Enterprise) – German energy utility (HQ: Essen, Germany) operating Europe-wide. Public case study shows E.ON is *migrating* from on-premises AEM to **AEM Sites on the Cloud (AEMaaCS)**, using AEM Sites (Cloud Service) and Adobe Target. India presence is unclear (mainly Europe), careers page: [eon.com/en/careers](https://www.eon.com/en/careers). Migrating from AEM 6.5: **Yes** (moving to Cloud Service). Edge Delivery: not mentioned. Likely roles include **AEM Developers/Engineers, Java Software Engineers, Platform Engineers**. Current hiring in India: unknown (likely minimal). Visa sponsorship: unclear. This is an attractive large-scale transformation (70+ sites across 14 countries) using modern cloud CMS and personalization. **Priority:** 8/10.

- **Cox Communications** (Telecommunications, Enterprise) – U.S. cable/ISP (HQ: Atlanta, GA). An Adobe case study confirms Cox migrated its web properties onto **AEM as a Cloud Service**, leveraging AEM Sites (Cloud Service), Experience Manager Assets, Adobe Campaign, Adobe Target, and Analytics. India presence: none (U.S.-centric), careers: [jobs.coxenterprises.com](https://jobs.coxenterprises.com/). Migrating from AEM 6.5: **Yes** (to AEMaaCS). Edge Delivery: not cited. Typical jobs: **AEM/Java Developers, Digital Content Engineers, DevOps Engineers**. Hiring in India: No. Visa: likely U.S.-only. Cox’s digital transformation yielded 50% of sales online; it’s a mature AEM Cloud user with strong Adobe integration. **Priority:** 8/10.

- **Workday, Inc.** (Enterprise Software, Product) – SaaS HR/Finance software (HQ: Pleasanton, CA). Summit 2025 talks detail Workday’s digital team choosing **AEM as a Cloud Service** from the start, using AEM Sites (Cloud), **Content Fragments**, **Experience Fragments**, and planning to leverage **Edge Delivery Services**. They also integrate Adobe Target and Real-Time CDP (AEP) for personalization. India presence: yes (development centers in Pune/Mumbai), careers: [workday.com/en-in/company/careers](https://www.workday.com/en-in/company/careers/). Migrating from 6.5: **Yes** (moving existing AEM to Cloud). Edge Delivery: **Yes (planned)**. Roles: **AEM Full-Stack Developers, Frontend Engineers (React), Java/AEM Backend Engineers, DevOps Engineers**. Hiring in India: Active (large global company). Visa: likely supportive (global workforce). Workday’s modern AEM Cloud implementation and focus on headless components makes it highly attractive for seasoned AEM engineers. **Priority:** 9/10.

Roles: **Front-End/Full-Stack Engineers (JavaScript/CSS), AEM Developers, Solution Architects**. Hiring in India: No. Visa: N/A. Attractive because it’s a cutting-edge Adobe Commerce/AEM project (headless, high performance) demonstrating EDS and Universal Editor use. **Priority:** 7/10.
